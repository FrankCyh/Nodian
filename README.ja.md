# Nodian

YAML frontmatter の双方向リレーションを自動同期する [Obsidian](https://obsidian.md) プラグインです。

あるファイルのフィールドに wikilink を追加すると、プラグインがターゲットファイルの対応フィールドにバックリンクを自動で書き込みます。リンクを削除すると、バックリンクも自動で削除されます。

## 例

```
Person.md                          Mail.md
─────────                          ─────────
tags: [Person]                     tags: [Mail]
Mail: [[hello@example]]     →     Person: [[Alice]]        ← 自動生成
```

## 機能

- **自動同期** — 一方のファイルでリンクを追加・削除すると、もう一方が即座に更新
- **リレーションペア** — どのフィールドがペアかを定義（例：`Mail ↔ Person`、`Artist ↔ Songs`）
- **タグベースのマッチング** — 各ペアにはタグが必須。フィールドとタグの両方が一致した場合のみ同期
- **表示名** — オプションで `title` フィールドをバックリンクの表示テキストとして使用
- **新規ファイル対応** — wikilink からファイルを作成すると、タグとバックリンクを自動追加
- **フル同期コマンド** — コマンドパレットからすべてのリレーションを手動同期
- **多言語対応** — 英語、日本語、繁体字中国語
- **モバイル対応** — デスクトップ・モバイルの両方で動作

## インストール

### BRAT を使用（推奨）

1. Community Plugins から [BRAT](https://github.com/TfTHacker/obsidian42-brat) をインストール
2. BRAT → Add Beta Plugin → `AkiSantin/Nodian`

### 手動インストール

`main.js`、`manifest.json`、`styles.css` を以下にコピー：

```
<vault>/.obsidian/plugins/nodian/
```

Obsidian を再起動 → 設定 → Community Plugins → Nodian を有効化。

## はじめに

### ステップ 1：最初のペアを作成

任意の `.md` ファイルを開き、YAML フィールドに wikilink を追加します：

```yaml
---
tags: [Person]
Mail: "[[hello@example]]"
---
```

プロパティ名を右クリック → **双方向リレーションを設定** を選択。

モーダルで対応フィールド（例：`Person`）とタグを設定し、**保存** を押します。

### ステップ 2：確認

`hello@example.md` を開くと、以下が表示されるはずです：

```yaml
---
tags: [Mail]
Person: "[[Alice]]"
---
```

これで完了です。以降、すべてのファイルの `Mail` フィールドがターゲットファイルの `Person` フィールドと自動同期されます。

### ステップ 3：フル同期（既存の Vault）

Vault に既存のリレーションがある場合、一度フル同期を実行します：

コマンドパレット（`Cmd/Ctrl+P`）→ **すべての双方向リレーションを同期**

すべてのファイルをスキャンし、不足しているバックリンクを補完します。

## 使い方

### リンクの追加

ペア設定済みのフィールドに wikilink を追加すると、ターゲットファイルにバックリンクが自動で表示されます。

```yaml
# Artist.md に入力：
Songs: "[[Blue Sky]]"

# プラグインが Blue Sky.md に自動生成：
Artist: "[[Artist Name]]"
```

### リンクの削除

フィールドから wikilink を削除すると、ターゲットファイルのバックリンクも自動で削除されます。

### 複数リンク

フィールドには複数の wikilink を含めることができます：

```yaml
Songs:
  - "[[Blue Sky]]"
  - "[[Red Moon]]"
  - "[[Green Field]]"
```

各ターゲットファイルにそれぞれバックリンクが作成されます。1つのリンクを削除しても、そのターゲットのみに影響します。

### セルフリレーション

フィールドは自分自身とペアにすることもできます：

```
Related ↔ Related
```

A.md に `Related: [[B]]` を追加すると、B.md に `Related: [[A]]` が自動で追加されます。

### 表示名

デフォルトでは、バックリンクはファイル名のみを使用します：`[[my-artist-id]]`。

`title` フィールドを表示テキストとして使うには、設定で **title を表示名として使用** を有効にしてください。バックリンクは `[[my-artist-id|Some Artist Name]]` のように表示されます（ソースファイルの `title` frontmatter フィールドの値を使用）。`title` フィールドのみが使用されます — `aliases` は参照されません。

この設定を切り替えても、既存のバックリンクは自動更新されません。変更後、コマンドパレットから **すべての双方向リレーションを同期** を実行して、Vault 全体のバックリンクを更新してください。

### 新規ファイルの作成

まだ存在しないファイルにリンクした場合：

```yaml
Mail: "[[new-contact]]"
```

`new-contact.md` が作成されると（例：Obsidian でリンクをクリック）、プラグインが：
1. 新しいファイルに適切なタグを自動追加
2. バックリンクを自動で書き込み

## タグ

すべてのリレーションペアには 2 つのタグ — **タグ A** と **タグ B** — があり、フィールド A とフィールド B に対応します。タグは**必須**であり、省略できません。

### タグの仕組み

プラグインは以下の **両方の条件** が満たされた場合にのみ同期します：

1. ソースファイルにペアのフィールド名と一致するフィールドがある
2. ソースファイルのタグがそのペアの対応するタグと一致する

これにより、誤ったターゲットへの同期を防ぎます。例えば、`Release` ファイルと `Song` ファイルの両方に `Artist` フィールドがある場合、タグによって各ペアが正しいファイルタイプでのみ動作します — `Release` タグのファイルは `Artist ↔ Release` ペアを使用し、`Song` タグのファイルは `Artist ↔ Song` ペアを使用します。

### タグの自動割り当て

- コンテキストメニューからペアを作成する際、ソースファイルのタグが自動的に使用されます。
- wikilink から新規ファイルが作成される際、プラグインはペア定義から適切なタグを自動で割り当てます。

## 設定

設定 → Nodian を開きます。

### リレーションペア

フィールドペアの追加・編集・削除。各ペアは 2 つのフィールド名と 2 つのタグを定義し、双方向にリンクされます。

| 設定 | デフォルト | 説明 |
|------|-----------|------|
| 自動同期 | オン | 編集時にバックリンクを自動同期 |
| title を表示名として使用 | オフ | バックリンクに `title` フィールドを表示テキストとして使用。変更後はフル同期を実行してください。 |
| デバッグモード | オフ | 開発者コンソールに詳細ログを出力（`Cmd/Ctrl+Option+I`、`[YBR]` でフィルタ） |

## ペアの例

各ペアには設定でタグ A とタグ B も設定されます。以下の表はフィールドのペアのみを表示しています — タグはペア作成時に設定またはコンテキストメニューから指定します。

音楽 Vault：

| フィールド A | ↔ | フィールド B |
|-------------|---|-------------|
| Artist | ↔ | Release |
| Artist | ↔ | Tracks |
| Composer | ↔ | Works |
| Label | ↔ | Releases |
| Related | ↔ | Related |

企業/CRM Vault：

| フィールド A | ↔ | フィールド B |
|-------------|---|-------------|
| Mail | ↔ | Person |
| Mail | ↔ | Domain |
| Mail | ↔ | Account |
| Service | ↔ | Account |

## 注意事項

- **初回使用前に Vault をバックアップしてください。** このプラグインは YAML frontmatter を直接変更します。ペアで定義されたフィールドのみを操作しますが、複雑なカスタム YAML がある場合、予期しないフォーマット変更が発生する可能性があります。
- **ファイルの削除** では、そのファイルを指すバックリンクは削除されません — これらは未解決リンクになります（偶発的なデータ損失を防ぐための設計上の選択です）。
- **ファイルのリネーム** は Obsidian の組み込みリンクアップデータが処理します — プラグインによる追加操作は不要です。
- **重複ペア**（例：`A ↔ B` と `B ↔ A`）は冗長です — 1 つのペアで双方向をカバーします。
- **重複するベース名** — 異なるフォルダに同名のファイルがあると、同期ターゲットが不正確になる可能性があります。一意のファイル名を使用してください。
- **システムフィールド**（`title`、`aliases`、`tags`、`cssclasses`、`publish` など）はリレーションフィールドとして使用できません。
